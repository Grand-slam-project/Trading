# Chatbot Routing Guard And Compound QA Design

## Goal

챗봇이 자산군/범주만 있는 질문을 종목 조회 실패로 처리하지 않고 확인 질문으로 응답하게 하며, 가격과 뉴스/공시/전망이 섞인 정보성 복합 질문은 여러 도구 결과를 묶어 답변하게 한다.

## Scope

포함:
- `코인 시세`, `주식 뉴스`, `공시 보여줘`, `뉴스 알려줘`, `코인 전망`처럼 종목 또는 검색 대상이 부족한 질문의 확인 응답
- 가격 + 뉴스, 가격 + 공시, 가격 + 전망 복합 질문 분해
- 복합 질문 결과의 단순 템플릿 조합
- 관련 백엔드 단위 테스트

제외:
- 주문 생성/승인/pending action 흐름 변경
- `삼성전자 1주 사줘 그리고 뉴스 알려줘` 같은 주문 포함 복합 질문 처리
- 프론트 UI 변경
- LLM 기반 복합 계획 생성

## Routing Guard

사용자가 종목명 없이 자산군 또는 범주만 입력한 경우 내부 조회 도구를 바로 실행하지 않는다.

예상 응답:
- `코인 시세 알려줘` → `어떤 코인의 시세를 볼까요? 예: BTC, ETH, XRP`
- `코인 전망 어때` → `어떤 코인의 전망을 볼까요? 예: BTC, ETH, XRP`
- `주식 뉴스 보여줘` → `어떤 종목이나 시장 뉴스를 볼까요? 예: 삼성전자 뉴스, 엔비디아 뉴스`
- `공시 보여줘` → `어떤 종목의 공시를 볼까요? 예: 삼성전자, SK하이닉스`
- `뉴스 알려줘` → `어떤 종목이나 시장 뉴스를 볼까요? 예: 삼성전자 뉴스, 비트코인 뉴스`

가드는 `run_chatbot_tool()`의 도구 라우팅 초반에 적용한다. 주문 요청은 기존 안전 흐름을 먼저 유지한다.

## Compound QA

정보성 복합 질문은 다음 조합만 처리한다.

- 가격 + 뉴스
- 가격 + 공시
- 가격 + 전망

예:
- `삼성전자 현재가랑 최신 뉴스 알려줘`
- `삼성전자 가격이랑 공시 요약해줘`
- `테슬라 가격 어때 전망도 알려줘`
- `비트코인 가격이랑 뉴스 알려줘`

처리 방식:
1. 종목명을 추출한다.
2. 가격 도구를 실행한다.
3. 보조 의도에 따라 `search_web()` 또는 `get_asset_outlook()`을 실행한다.
4. 두 응답을 `현재가`와 `추가 확인` 섹션으로 묶어 반환한다.
5. `data.source`는 `COMPOUND_INFO`로 기록하고, 하위 결과를 `price`와 `secondary`에 저장한다.

## Safety Rules

- 주문 키워드가 있는 경우 복합 정보 분해를 실행하지 않는다.
- 매매 제안 생성 및 실거래 실행 흐름은 변경하지 않는다.
- 복합 질문의 보조 결과가 실패하면 가격 결과는 유지하고, 보조 결과 실패 안내를 붙인다.
- 확인 응답은 사용자가 다음에 무엇을 입력해야 하는지 예시를 포함한다.

## Testing

필수 테스트:
- 자산군/범주만 있는 질문이 확인 응답으로 라우팅되는지
- 종목이 있는 가격+뉴스 질문이 가격 도구와 뉴스 도구를 모두 호출하는지
- 종목이 있는 가격+공시 질문이 가격 도구와 공시 도구를 모두 호출하는지
- 종목이 있는 가격+전망 질문이 가격 도구와 전망 도구를 모두 호출하는지
- 주문 포함 복합 질문은 기존 주문 안전 흐름을 유지하는지
- 전체 챗봇 테스트가 통과하는지

검증 명령:

```bash
PYTHONPATH=. pytest backend/tests/test_chatbot_*.py tests/backend/test_chatbot_*.py -q
```
