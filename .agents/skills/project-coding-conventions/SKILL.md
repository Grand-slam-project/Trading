# Project Coding Conventions

이 스킬은 이 저장소에서 코드 수정, 문서 수정, 테스트, 커밋을 수행할 때 적용하는 로컬 규칙입니다.

## 커밋 메시지

- Conventional Commit prefix는 영문으로 유지합니다.
- 콜론 뒤 설명은 한글로 작성합니다.
- 형식: `<type>: <한글 설명>`
- 예시:
  - `docs: AssetDetail 리팩토링 상태 기록`
  - `refactor: 종목 상세 공통 모델 분리`
  - `test: 종목 상세 모델 테스트 추가`
  - `fix: 자연어 주문 분석 의도 오분류 방지`

## 작업 보고

- 설명, 계획, 결과 보고는 한국어로 작성합니다.
- 검증 결과는 실행한 명령과 통과/실패 여부를 함께 기록합니다.
- 남은 리스크가 있으면 숨기지 않고 후속 작업으로 분리합니다.
